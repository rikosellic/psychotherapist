# 知识梳理与提取（Knowledge Organization and Extraction）

本 skill 用于将一本心理学/心理治疗相关书籍的原始内容（PDF 提取文本或 OCR Markdown）加工为结构化、可被 AI agent 检索和引用的 Markdown 读书笔记，输出到 `skills/<书籍名称>/` 目录下。

## 前置条件

在运行本 skill 之前，需要先使用 `tools/pdf-analyzer` 将原始 PDF 提取为文本：

- **可复制文字 PDF**：使用 `pdf-extract` 路线，输出 Markdown。
- **扫描版/图片版 PDF**：使用 `pdf-ocr` 路线，输出 OCR Markdown。

提取后的文件通常位于 `output/markdown/` 或 `output/ocr-markdown/` 目录下。

## 输入与输出

### 输入

以 `tools/pdf-analyzer` 输出的 Markdown 文件作为输入。文件路径示例：

- `output/markdown/<书名>.md`（可复制文字 PDF）
- `output/ocr-markdown/<书名>.md`（OCR 扫描版）

### 输出

在 `skills/<书籍名称>/` 下生成以下文件结构：

```
skills/<书籍名称>/
├── book/                          # 原始 PDF 存放处
│   └── <书名>.pdf
├── <书籍名称>_index.md            # 索引导航文件
├── <书籍名称>_01_04.md            # 章节文件（覆盖第 1-4 章）
├── <书籍名称>_05_07.md            # 章节文件（覆盖第 5-7 章）
└── ...                            # 更多章节文件
```

## 工作流程

### 步骤 1：通读与理解全书结构

1. 完整阅读输入的 OCR Markdown 文件，理解全书章节划分、逻辑脉络和核心主题。
2. 识别书的自然分段：哪些章节属于同一主题模块，哪些章节需要独立成文件。
3. 标记所有心理学专业术语，确保后续提取时保留对应的英文原文。

### 步骤 2：规划文件拆分

根据章节内容的组织性和结构性，决定如何将连续章节合并到同一个文件中。原则：

- **内容密度**：如果一章内容非常丰富（如详细介绍多个技能模块），可以单独成文件。
- **主题关联**：同属一个主题模块的连续章节优先合并（如"评估与诊断"涵盖 3-4 章，合并为一个文件）。章节内容或流派有所相关，值得比较阅读的（如“第2章精神分析”与“第3章阿德勒”，可以合并为一个文件）。我们需要把一本完整的书按照其内部逻辑或者上述的章节关联进行一些合并，尽量避免一整本书里的每个章节都单独成为一个md文件（偶尔有些很短的书或者有些重要的、归纳性的章节，可以单独成为一个文件）。
- **篇幅均衡**：文件中每个章节的内容量大致均衡，避免某个章节过长或过短。提取后章节的字数量可以是原文章节的20%左右。但如果章节有特殊含义，或特殊需求，也可以破例较长或较短。
- **参考已有范例**：可以参考 `skills/Counseling and Psychotherapy Theories in Context and Practice/` 的文件拆分思路，但不要照搬其内部结构。

### 步骤 3：逐文件提取内容

对每个规划好的章节文件，按以下要求提取：

#### 核心原则

1. **压缩提炼，而非扩写复述**：输出应明显短于原文，起到精简总结和知识提炼作用。优先保留概念框架、关键定义、操作步骤、临床判断点和实践注意事项；删去重复论述、铺垫性文字、低信息密度例子和可由标题自然推知的内容。
2. **不编造内容**：所有知识点必须在原文中有依据。可以有理解性的组织和归纳，但不能添加书中没有的论断、案例或解释。
3. **结构服务于理解**：章节顺序和作者的主要逻辑脉络应被尊重，但文件内部的组织结构不必机械复刻原书或参考范例。应根据本书实际内容自行决定层级、模块和小节，使没看过这本书的人也能理清全书思路框架，读懂并能够运用其中知识与技术实践。
4. **保留英文术语**：对于心理学专业术语，如果原文中包含相应英文，提取内容中必须保留英文原文。格式示例：`认知行为疗法（Cognitive Behavioral Therapy，CBT）`、`边缘型人格障碍（Borderline Personality Disorder，BPD）`。

#### 文件格式规范

每个章节文件必须包含以下部分：

**文件头部**

```markdown
# <书籍名称>_<章节范围>

来源：`<OCR Markdown 文件路径>`
抽取范围：<原书章节范围，如"第 1-4 章">，**<章节英文标题>**。
用途：<一句话说明该文件在 agent 使用中的角色>。
```

**正文内容**

- 使用层级标题组织内容（`##` 章、`###` 节、`####` 小节）。
- 正文以要点列表（`-`）为主，每点一行，简洁直接。
- 对关键概念、定义、操作步骤、临床技巧，保留足够的细节使 agent 可以引用；对重复解释、长案例和枝节说明进行概括。
- 保留原书中的重要表格、框架、流程图，用 Markdown 表格或列表压缩表达。
- 在章节标题后可以添加简短的"章节主线"段落（3-5 句），概括该章的核心论点。

**文件末尾（可选）：综合分析**

如果该文件覆盖的几章内容之间存在值得关联或对比的关系，在文件末尾添加 `## 综合分析` 部分，简要说明这几章之间的逻辑关系和互补性。

**文件末尾（必需）：Agent 使用建议**

每个文件末尾必须包含 `## Agent 使用建议` 部分，说明：
- 该文件适合在什么场景下被 agent 引用。
- 哪些内容适合直接用于个案概念化、干预建议或诊断参考。
- 与其他文件的关联（如"需要结合 XX_index.md 中的 YY 章节"）。

格式示例：

```markdown
## Agent 使用建议

- **适用场景**：当 agent 需要理解 CBT 的认知模型、行为激活技术或自动思维的识别与干预时使用。
- **可直接引用**：自动思维的特征列表、认知歪曲分类、行为实验的设计步骤。
- **需要结合**：`XX_index.md` 中的治疗计划章节，用于将概念化转化为具体干预方案。
- **注意**：本章的技术描述基于标准 CBT 框架，应用于不同文化背景的来访者时需进行文化适配。
```

### 步骤 4：生成索引文件

创建 `<书籍名称>_index.md`，格式如下：

```markdown
# <书籍名称>_index

来源：`<OCR Markdown 文件路径>`
原书：<作者>，*<书名>*。
用途：供 agent 使用中文快速查询"<书籍主题>"框架。……（说明提取方式）

## 文件导览

### <文件名_1>.md

覆盖第 X 至第 Y 章：<章节标题>。

适合查询：
- <知识点 1>
- <知识点 2>
- ...

### <文件名_2>.md

覆盖第 X 至第 Y 章：<章节标题>。

适合查询：
- ...
```

### 步骤 5：质量检查

完成所有文件后，对照以下检查清单自检：

- [ ] 所有知识点是否在原文中有依据，无编造内容？
- [ ] 输出是否明显短于原文，发挥了精简总结和知识提炼作用？
- [ ] 是否尊重原书章节顺序和主要逻辑脉络，同时根据本书内容设计了清晰的内部组织结构？
- [ ] 所有心理学专业术语是否保留了英文原文？
- [ ] 每个文件末尾是否包含 `## Agent 使用建议`（包含index文件的末尾）？
- [ ] 索引文件是否准确描述了每个文件的内容和适用场景？
- [ ] 文件拆分是否合理，篇幅是否均衡？
- [ ] 语言是否简洁，无冗余废话？
- [ ] 生成文件的文件名是否符合之前其他文件夹中的命名格式？参考格式例如：`ICD-11精神、行为与神经发育障碍临床描述与诊断指南 B5_05_08_焦虑强迫应激分离`

## 参考范例

参考范例位于 `skills/Counseling and Psychotherapy Theories in Context and Practice/`，请在处理每本书之前先阅读该目录下的文件，理解其索引写法、语言密度、标题层级和 `Agent 使用建议` 的表达方式。

范例只用于参考，不是必须遵循的固定模板。处理新书时，请根据你从原书中提取出的内容自行决定文件内部的组织结构：哪些内容应作为主线框架、哪些内容应合并为技能模块、哪些内容应拆为步骤或表格。最终目标是让没看过这本书的人也能通过你的组织结构理清思路框架，读懂这本书，并掌握其中的知识与技术实践。

重点参考文件：
- `Counseling and Psychotherapy Theories in Context and Practice_index.md`：索引文件格式。
- `Counseling and Psychotherapy Theories in Context and Practice_01_心理治疗与咨询基础.md`：章节文件格式，特别是"章节主线"和"Agent 使用建议"的写法。

## 注意事项

1. **不要跳过 Agent 使用建议**：这是 agent 后续检索和引用该书内容的关键入口。
2. **不要写成逐段复述**：目标是让没看过这本书的 agent 能理清思路框架并掌握其中知识与技术实践，风格应偏向"可检索的知识库"而非"原文缩写版"。
3. **尊重原书主线**：不要重新编排原书章节顺序；但章节文件内部的小节、模块、表格和步骤，可以根据内容理解重新组织，以帮助读者建立框架。
4. **处理多册书籍**：如果一本书分上下册，分别放在独立目录中，如 `skills/DBT情绪调节手册（上）/` 和 `skills/DBT情绪调节手册（下）/`。
