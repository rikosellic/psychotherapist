# 个案概念化

本 skill 用于根据心理咨询或心理治疗材料，生成专业中文个案概念化报告，并在报告基础上为治疗师提供后续干预指导。

当用户提出以下需求时，应使用本 skill：

- 根据来访者治疗记录进行个案概念化
- 撰写、整理或优化个案概念化报告
- 根据咨询记录分析治疗难点
- 识别治疗师在既往会谈中已经使用过的干预技术
- 为治疗师提供后续干预建议
- 将个案材料整理成某一流派的概念化报告
- 在督导或个案研讨中理解来访者问题的形成与维持机制

## 资料使用优先级

按以下顺序使用本地参考资料：

1. `references/个案概念化理论/`
   - 这是个案概念化思路的主要来源。
   - 该文件夹中的两本核心书用于指导如何提取材料、形成假设、识别维持机制，并将概念化连接到治疗含义。
2. `references/个案概念化模板/`
   - 这是个案概念化报告格式的主要来源。
   - 其中不同模板可能属于不同临床流派。若用户指定流派，应优先匹配对应模板。
3. `references/干预/`
   - 这是后续治疗干预指导的主要来源。
   - 应优先使用与概念化报告流派一致的干预资料。
4. `references/index.md`
   - 这是资源使用说明索引，用于快速确认各文件夹的职责。

不要把三类资料混用。理论资料负责“如何理解个案”，模板资料负责“报告如何呈现”，干预资料负责“识别治疗师之前运用了哪些干预技术并指导治疗师接下来如何工作”。

## 工作流程

### 1. 明确任务与材料状态

先判断：

- 用户需要完整报告、简版概念化、指定流派报告，还是只需要干预指导。
- 用户提供的是初访记录、咨询记录、治疗逐字稿、评估摘要、督导材料，还是报告草稿。
- 用户是否指定了理论取向或报告模板。
- 材料中是否出现风险线索，例如自杀、自伤、伤人、虐待、精神病性症状、躁狂、严重物质使用、进食障碍风险、解离或保护性议题。

如果材料不足以形成完整报告，仍可以生成谨慎版本，但必须在“需进一步评估的信息”中明确标出缺失内容。

### 2. 提取临床相关信息

将原始材料整理为以下信息：

- 基本信息与材料来源
- 主诉与求助动机
- 当前症状、困扰和功能受损
- 诱发事件与近期压力源
- 问题的发展过程
- 重要成长史、家庭史、依恋经验和关系经验
- 既往治疗史、诊断史和用药史，如材料中有提及
- 治疗师在记录中已经使用过的干预技术、治疗意图和来访者反应，如材料中有提及
- 咨询过程中的互动模式、阻抗、联盟变化和情绪反应
- 资源、优势与保护因素
- 风险线索与伦理注意事项

必须区分事实与假设。除非材料中有直接依据，不要把推测出的动机、诊断、创伤机制、人格模式或家庭动力写成确定事实。

### 3. 基于理论资料形成概念化

使用 `references/个案概念化理论/` 作为主要推理基础。个案概念化需要解释：

- 来访者为什么在此时求助。
- 来访者表层问题是什么。
- 哪些易感因素、发展经验或关系经验可能塑造了当前问题。
- 哪些近期因素诱发或激活了当前困难。
- 哪些认知、情绪、行为、关系、身体反应、系统和环境机制维持了问题。
- 哪些资源、优势和保护因素可以被治疗利用。
- 哪些假设仍不确定，需要进一步评估。

必要时，可使用以下最低限度框架组织概念化：

- 表层问题：症状、冲突、功能受损和求助议题
- 形成因素：发展史、重要关系、学习经验、创伤或压力背景
- 诱发因素：近期事件、阶段任务、关系变化或危机
- 维持机制：认知、情绪、行为、关系、身体反应和环境反馈循环
- 核心假设：核心信念、核心冲突、关系模式、情绪图式或自我组织方式
- 资源与保护因素：能力、支持、动机、价值和既往有效经验
- 治疗含义：治疗目标、治疗焦点、风险点和治疗关系中的预期难点

### 4. 选择或匹配报告模板

使用 `references/个案概念化模板/` 决定报告结构。

如果用户指定流派，在本地有对应模板时使用对应模板。如果用户没有指定流派，应根据材料特征选择最合适的模板，并简要说明选择理由。

默认匹配逻辑：

- 认知行为取向：当材料重点涉及症状、自动思维、回避、应对行为和可测量目标时优先选择。
- 心理动力取向：当材料重点涉及重复关系模式、防御、情感调节、早期关系、移情或无意识冲突时优先选择。
- 家庭/系统取向：当材料重点涉及家庭结构、互动循环、角色期待、代际模式、伴侣或家庭冲突时优先选择。
- 依恋/创伤知情取向：当材料重点涉及创伤、安全感、解离、羞耻、依恋损伤、身体化威胁反应或关系安全时优先选择。
- 整合式取向：当多个机制都很突出，或用户要求完整专业报告时优先选择。

如果没有找到明确匹配的本地模板，使用整合式个案概念化报告结构，并说明报告采用通用整合式结构。

### 5. 生成个案概念化报告

默认输出语言为中文。

默认报告结构：

1. 个案基本信息与材料来源
2. 主诉与当前主要困扰
3. 治疗记录摘要（含既往干预技术识别）
4. 重要发展史与关系经验
5. 当前问题表现与功能影响
6. 个案概念化
7. 风险因素、资源与保护因素
8. 治疗目标
9. 后续治疗难点
10. 干预路径与治疗师建议
11. 需进一步评估的信息

可以根据所选模板调整标题，但除非用户要求简化，否则应覆盖上述内容域。

写作要求：

- 写成专业个案报告草稿，而不是日常解释或教学讲义。
- 使用清晰的临床语言，但避免不必要的术语堆叠。
- 在“治疗记录摘要”中概括会谈进程，并单独识别治疗师已经使用的干预技术；写明技术名称、对应材料依据、可能的治疗意图和来访者反应。
- 对既往干预技术的识别必须基于记录中的治疗师话语、任务布置、互动方式或会谈结构；材料不足时写“记录中未能明确识别”，不要凭治疗取向倒推。
- 对不确定内容使用“可能”“提示”“可假设”“仍需进一步评估”等表达。
- 不编造缺失的人口学信息、诊断、咨询次数、家庭史、创伤史或治疗结果。
- 不过度断言因果关系，优先使用概念化语言而非绝对判断。
- 只有在材料支持时，才写入临床相关的阴性信息。

### 6. 在报告基础上生成干预指导

干预指导必须基于已经完成的个案概念化报告，而不是直接从孤立症状跳到技术建议。

使用 `references/干预/` 识别记录中已经出现的干预技术，并指导面向治疗师的后续建议。优先选择与报告模板流派一致的干预资料。若采用整合式报告，则选择能够直接对应报告中维持机制的干预方向。

干预指导应组织为：

- 短期工作重点：安全、联盟、评估、稳定化、症状缓解和治疗框架
- 中期工作重点：核心维持机制、情绪/认知/关系模式、技能训练或经验加工
- 长期工作重点：人格或关系模式改变、复发预防、意义整合和生活结构调整
- 治疗师需要注意的难点：联盟风险、回避/阻抗、情绪过载、边界议题、反移情和风险升级
- 可使用的技术方向：只纳入被所选流派和个案概念化支持的技术
- 观察指标：如何判断干预是否有效，以及何时需要调整治疗策略

每条干预建议都应说明：

- 它针对概念化中的哪个机制。
- 它为什么适合来访者当前表现。
- 治疗师需要观察什么。

### 7. 安全与临床边界

本 skill 用于生成专业报告草稿和督导取向的治疗建议，不替代正式诊断、急危机处理、医学建议、法律判断或现场临床督导。

当材料中出现高风险内容时：

- 单独加入风险部分。
- 建议治疗师立即进行结构化风险评估，并按照当地专业、伦理和法律要求寻求督导、转介或危机处理。
- 不提供可能促进自伤、伤人、胁迫或逃避照护的具体操作。

关于诊断：

- 只有当用户明确要求，或材料已经提供充分依据时，才使用诊断语言。
- 优先使用“诊断假设”或“鉴别方向”，避免在证据不足时给出确定诊断。

## 输出模式

如果用户没有指定模式，默认使用“完整版报告”。

- `完整版报告`：完整个案概念化报告加干预指导。
- `简版概念化`：简要概念化和关键治疗含义。
- `督导讨论版`：重点呈现假设、治疗过程、治疗师困境和下一次咨询策略。
- `指定流派版`：按照用户指定的理论取向和对应模板生成。
- `干预指导版`：仅当用户已经有概念化报告，并希望获得下一步治疗建议时使用。

## 最终回复检查清单

回复用户前，确认输出已经做到：

- 使用 `个案概念化理论` 文件夹作为概念化推理基础。
- 使用 `个案概念化模板` 文件夹作为报告结构依据。
- 使用 `干预` 文件夹识别既往干预技术，并作为报告之后的治疗指导依据。
- 区分事实、推论、假设和缺失信息。
- 在用户未指定流派时，说明选择的模板或取向。
- 在“治疗记录摘要”中包含治疗师已使用干预技术的识别；若材料不足，明确说明无法识别。
- 在概念化之后包含治疗难点和治疗师指导。
- 当材料涉及风险或信息不足时，明确提示风险和进一步评估需要。
