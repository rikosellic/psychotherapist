# 个案概念化

本 skill 用于根据心理咨询或心理治疗材料，生成专业个案概念化分析，并为治疗师提供后续干预指导。

## 知识来源声明

本 skill 以 `skills/` 目录为**唯一知识来源**。知识库包含四部分：

| 模块 | 路径 | 内容 |
|------|------|------|
| 个案概念化 | `skills/个案概念化/` | CBT 个案概念化（Persons, 2008）、心理动力学个案概念化（Cabaniss, 2013） |
| 综合理论 | `skills/Counseling and Psychotherapy Theories in Context and Practice/` | 13 种咨询与心理治疗理论，各章均含个案概念化模板与干预技术 |
| 干预 | `skills/干预/` | 12 本干预手册/指南（DBT、CBT、IPT、SFBT、正念、团体、访谈等） |
| 诊断 | `skills/诊断/` | DSM-5 中文版、ICD-11 临床描述与诊断指南 |
| 模板 | `skills/模板/` | 5 个模板：4 个流派报告模板 + 1 个干预指导模板（`.md` markdown解析版 + `.docx` 原件） |

**不得引用以下目录中的任何资料**：`references/`、`colleague-sunfu2/`、`tmp/`、或 `skills/` 之外的任何路径。

## 三级知识读取体系

所有知识按深度分为三级，agent 必须严格遵循读取纪律：

| 级别 | 定义 | 内容 | 何时使用 |
|------|------|------|----------|
| **L1 索引级** | 导航与框架 | `*_index.md` 文件，含章节导览、核心概念速览、Agent 使用建议 | 每次任务的起点——确定需要读哪些章节文件 |
| **L2 章节级** | 结构化知识 | 章节合并 `.md` 文件（如 `_01_03_总论与三步法.md`），含理论框架、操作方法和临床要点 | **默认分析深度**——绝大多数任务在此完成 |
| **L3 原文级** | 完整原文 | `book/` 目录下的完整书籍 `.md` 文件，含逐页原文 | **仅在明确触发条件下进入** |

### L3 原文触发条件

满足以下任一条件时，方可从 L2 进入 L3 读取 `book/` 原文：

1. 需要精确的技术步骤、练习引导语或表格字段
2. 需要核对诊断标准的精确措辞（边界鉴别场景）
3. 需要特定疗法的会话脚本或示范对话
4. L2 章节文件中的描述不足以解决用户的具体技术问题
5. 用户明确追问"原文怎么说的""具体步骤是什么""书里原话是什么"

### 不应进入 L3 的场景

以下场景必须停在 L2，不得读取 `book/` 原文：

1. 常规个案概念化分析
2. 理论框架解释或流派概述
3. 一般性干预方向建议
4. 症状模式识别或维持机制推断

**诊断特殊规则**：常规症状群识别和鉴别方向使用 L2 章节文件（DSM-5/ICD-11 章节文件已含诊断标准概要）；仅在边界案例、诊断争议或用户追问精确诊断标准时进入 L3。

## 材料处理

### 用户传入 PDF 材料时

1. 使用 `tools/pdf-analyzer` 提取 PDF 内容：
   - 可复制文字 PDF：`uv run pdf-extract "file.pdf" -o "tools/pdf-analyzer/tmp/extracted.md"`
   - 扫描版 PDF：`uv run pdf-ocr "scanned-file.pdf" -o "tools/pdf-analyzer/tmp/extracted.md"`
2. 读取提取后的 Markdown 文件，进入分析流程。
3. 分析完成后可清理临时文件。

### 用户传入文本/Markdown/对话记录时

直接读取材料内容，进入分析流程。

## 工作流程

### 1. 明确任务与材料状态

先判断：

- 用户提供的是初访记录、咨询记录、治疗逐字稿、评估摘要、督导材料，还是报告草稿。
- 用户是否指定理论取向或诊断系统。
- 材料中是否出现风险线索：自杀、自伤、伤人、虐待、精神病性症状、躁狂、严重物质使用、进食障碍风险、解离或保护性议题。

### 2. 提取临床相关信息

将原始材料整理为：

- 基本信息与材料来源
- 主诉与求助动机
- 当前症状、困扰和功能受损
- 诱发事件与近期压力源
- 问题的发展过程
- 重要成长史、家庭史、依恋经验和关系经验
- 既往治疗史、诊断史和用药史
- 治疗师已经使用过的干预技术、治疗意图和来访者反应
- 咨询过程中的互动模式、阻抗、联盟变化和情绪反应
- 资源、优势与保护因素
- 风险线索与伦理注意事项

**必须区分事实与假设**。除非材料中有直接依据，不要把推测出的动机、诊断、创伤机制、人格模式或家庭动力写成确定事实。

### 3. 确定分析路径

**必须先读** `skills/个案概念化任务读取指南.md`，按以下决策链确定分析路径：

```
材料特征 → 查指南中的理论取向匹配规则
  → 确定主要理论取向
    → 读取对应书籍的 *_index.md（L1）
      → 按 index 中的「Agent 使用建议」决定读哪些章节文件（L2）
        → 以 L2 知识完成分析
          → [仅在 L3 触发条件下] 进入 book/ 原文
```

**无明确取向时**：先读 `skills/Counseling and Psychotherapy Theories in Context and Practice/Counseling and Psychotherapy Theories in Context and Practice_index.md`，扫描各理论核心特征，按材料呈现的主要症状/模式/关系特征自动匹配最相关的 1–2 个理论。

**干预技术识别时**：根据个案概念化中识别的维持机制，查指南中的干预匹配表，确定需要读取的干预书籍，按同样顺序（index → 章节文件）获取知识。

**涉及诊断时**：按症状群查指南中的诊断模块，确定读取 DSM-5 或 ICD-11，按 index → 章节文件顺序获取诊断标准概要。诊断仅作为假设和鉴别方向，不做确定诊断。

### 4. 形成概念化

以 L2 章节文件中的理论知识为基础，将提取的临床信息组织为概念化假设。个案概念化需要解释：

- 来访者为什么在此时求助
- 表层问题、功能损害和主要困扰是什么
- 哪些易感因素、发展经验或关系经验可能塑造了当前问题
- 哪些近期因素诱发或激活了当前困难
- 哪些认知、情绪、行为、关系、身体反应、系统和环境机制维持了问题
- 哪些资源、优势和保护因素可以被治疗利用
- 哪些假设仍不确定，需要进一步评估

必要时使用最低限度框架：表层问题 → 形成因素 → 诱发因素 → 维持机制 → 核心假设 → 资源与保护因素 → 治疗含义。

### 5. 生成干预指导

干预指导必须基于已完成的个案概念化，而不是直接从孤立症状跳到技术建议。

使用指南中的干预匹配表，从相关干预书籍的 L2 章节文件中获取知识。干预指导应包含：

- 短期工作重点：安全、联盟、评估、稳定化、症状缓解和治疗框架
- 中期工作重点：核心维持机制、情绪/认知/关系模式、技能训练或经验加工
- 长期工作重点：人格或关系模式改变、复发预防、意义整合和生活结构调整
- 治疗师需要注意的难点：联盟风险、回避/阻抗、情绪过载、边界议题、反移情和风险升级
- 可使用的技术方向：只纳入被所选流派和个案概念化支持的技术
- 观察指标：如何判断干预是否有效，以及何时需要调整治疗策略

每条干预建议都应说明：
- 它针对概念化中的哪个维持机制
- 它为什么适合来访者当前表现
- 治疗师需要观察什么

## 输出规则

### 两档输出决策

收到用户请求时，首先判断应使用哪一档输出：

| 条件 | 输出档位 | 行为 |
|------|---------|------|
| ① 用户提供了较完整的治疗记录或初访材料；**且** ② 用户明确要求生成个案概念化报告；**或** ③ 材料明显足以支撑认真分析 | **正式报告档** | 按流派模板生成结构化个案概念化报告 |
| 用户只是询问问题、要求一点分析、材料零散、或未要求正式报告 | **自由分析档** | 不套模板，按书中理论知识自由组织回答 |

### 正式报告档

#### 模板选择

从 `skills/模板/` 中按以下规则选择模板：

| 用户指定/材料特征 | 使用模板 |
|---|---|
| 明确要求 CBT 取向，或材料核心为自动思维、行为循环、症状监测 | `skills/模板/认知行为治疗记录.md` |
| 明确要求心理动力学或精神分析取向，或材料核心为关系模式、防御、移情、发展史 | `skills/模板/动力学个案报告提纲.md` |
| 明确要求家庭/系统取向，或材料核心为家庭互动、代际模式、三角化 | `skills/模板/家庭案例督导汇报提纲.md` |
| 未指定流派、多机制并存、或要求完整专业报告 | `skills/模板/中德班_案例报告格式.md`（默认） |
| 用户已有概念化报告，仅需干预指导；或用户明确要求"干预指导版" | `skills/模板/干预指导模板.md` |

可综合多个模板：当个案涉及多个层面时（如个体症状+家庭系统），以主要模板为骨架，补充其他模板中相关部分的写法。

#### 模板使用方式

- 模板 `.md` 文件为markdown解析版，与docx原件内容一致。
- 以模板的结构（标题层级、字段顺序）组织输出。
- 概念化内容来自 `skills/` 知识库的 L2 分析结果，填入模板结构。

### 自由分析档

不套模板时：
- 按 `skills/个案概念化任务读取指南.md` 的理论框架组织回答。
- 可使用最低限度框架（表层问题→形成因素→诱发因素→维持机制→核心假设→资源→治疗含义）。
- 可从 Counseling Theories 对应章节的"个案概念化模板"中提取简化的概念化结构。
- 仍然遵守反编造控制协议（来源标注、事实-推论-假设区分）。
- **可提供干预指导**：当用户问及后续治疗建议时，参照 `skills/模板/干预指导模板.md` 的 7 模块结构组织回答，但不强制套用模板标题（可按用户问题灵活裁剪）。

## 输出模式

以下模式仅在**正式报告档**中适用。若用户未指定模式，默认使用 `完整版报告`。

- `完整版报告`：完整个案概念化报告加干预指导。
- `简版概念化`：简要概念化和关键治疗含义。
- `督导讨论版`：重点呈现假设、治疗过程、治疗师困境和下一次咨询策略。
- `指定流派版`：按照用户指定的理论取向和对应模板生成。
- `干预指导版`：仅当用户已经有概念化报告，并希望获得下一步治疗建议时使用。

## 反编造控制协议

五层防线确保 agent 不编造知识：

### ① 来源必须可追溯

每条基于理论的断言必须在段末或节末标注来源。格式：

```
（来源：{书名}，{章节文件名}）
```

示例：
- `（来源：Cabaniss 心理动力学个案概念化，_04_08_描述功能.md）`
- `（来源：DBT 技能训练手册，_02_痛苦耐受技能.md）`
- `（来源：DSM-5 中文版，DSM-5_05_08_焦虑_强迫_创伤_分离障碍.md）`

### ② 推测定性声明

无法在 `skills/` 中找到直接理论支持的判断，必须标注：

> ⚠️ 此观点未在 skills 知识库中找到直接支持，以下为基于临床常识的推测。

### ③ 知识来源清单

每次分析输出末尾，列出本次实际使用的所有 `skills/` 文件路径，格式：

```
---
📚 本次分析使用的知识来源（均位于 skills/ 目录下）：
- skills/个案概念化任务读取指南.md
- skills/个案概念化/英文版-认知行为治疗个案概念化.../..._index.md
- skills/个案概念化/英文版-认知行为治疗个案概念化.../_05_07_预治疗阶段.md
- skills/干预/dbt_skills_training+manuel/dbt_skills_training+manuel_index.md
...
```

### ④ 禁止外部引用

不得引用 `references/`、`colleague-sunfu2/`、`tmp/` 或任何 `skills/` 外的资料。若某个知识点可能存在于被排除的目录中，不得自行推断其内容，而应标注"该领域资料不在当前知识库范围内"。

### ⑤ 事实-推论-假设三级区分

- **事实**：材料中直接呈现的信息（来访者原话、治疗师记录、可观察行为）
- **推论**：基于可观察模式、重复现象和理论框架的合理推断
- **假设**：需要进一步评估验证的暂定判断

对不确定内容使用"可能""提示""可假设""仍需进一步评估"等表达。不编造缺失的人口学信息、诊断、咨询次数、家庭史、创伤史或治疗结果。不过度断言因果关系。

## 安全与临床边界

本 skill 用于生成专业分析草稿和督导取向的治疗建议，不替代正式诊断、急危机处理、医学建议、法律判断或现场临床督导。

当材料中出现高风险内容时：
- 单独加入风险部分。
- 建议治疗师立即进行结构化风险评估，并按照当地专业、伦理和法律要求寻求督导、转介或危机处理。
- 不提供可能促进自伤、伤人、胁迫或逃避照护的具体操作。

关于诊断：
- 只有当用户明确要求，或材料已经提供充分依据时，才使用诊断语言。
- 优先使用"诊断假设"或"鉴别方向"，避免在证据不足时给出确定诊断。
