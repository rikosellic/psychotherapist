# 个案概念化任务读取指南

本文件用于指导 Agent 在执行“按某个理论生成个案概念化”任务时，决定优先读取哪些知识库内容，以及如何组合专门个案概念化书籍、理论补充章节、诊断手册和干预手册。

## 通用原则

阅读深度、事实忠实性、输出档位、来源标注和安全边界统一由 `guides/临床工作流程.md` 规定。本指南只负责把任务特征映射到理论取向、知识目录和章节文件。

1. 先判断用户指定的主要理论取向；若用户未指定，按材料特征自动匹配（见下方各模块匹配表）。
2. 根据本指南确定需要进入的知识目录和候选章节，具体读取顺序与深度服从临床工作流程的 L1/L2/L3 规则。
3. 生成个案概念化时，所选知识应能支持：
   - 该理论的核心概念与个案理解框架；
   - 该理论对应的临床评估重点；
   - 该理论下的维持因素、改变机制和干预方向；
   - `Counseling and Psychotherapy Theories in Context and Practice` 对应章节中的"个案概念化模板"作为概念化结构参考。

## 认知行为治疗取向

适用情况：

- 用户明确要求以认知行为治疗、CBT、认知治疗、行为治疗、认知行为个案概念化为主。
- 用户要求围绕自动思维、中间信念、核心信念、行为实验、维持循环、情境-想法-情绪-行为链条来组织个案。

读取顺序：

1. **优先读取（L1 → L2）**：

   `./knowledge/个案概念化/英文版-认知行为治疗个案概念化The_Case_FormulaTion_approaCh_To_CogniTi/`

   先读：
   `英文版-认知行为治疗个案概念化The_Case_FormulaTion_approaCh_To_CogniTi_index.md`

   再根据任务需要读取对应章节文件：
   - 完整 CBT 个案概念化 → `_01_个案概念化取向.md` + `_05_07_预治疗阶段.md`
   - 理论基础/不同理论视角 → `_02_04_理论基石.md`
   - 治疗关系或来访者不配合 → `_08_09_治疗关系与监测.md`
   - 会谈决策或治疗失败 → `_10_12_临床决策.md`

2. **再读取理论补充（L1 → L2）**：

   `./knowledge/Counseling and Psychotherapy Theories in Context and Practice/`

   先读：
   `Counseling and Psychotherapy Theories in Context and Practice_index.md`

   CBT 相关章节：
   `Counseling and Psychotherapy Theories in Context and Practice_07_09_行为认知行为与现实治疗.md`

   该文件用于补充 CBT 在咨询理论体系中的治疗目标、治疗关系、技术选择、变化机制，以及该章节内的"个案概念化模板"。

3. **干预技术补充**：

   a. **CBT 全流程实践指南**（适用于所有 CBT 个案，优先读取）：
      `./knowledge/干预/认知行为疗法(第三版)/`
      先读 `*_index.md`（L1），按 Agent 使用建议读取对应章节。该书籍用于补充：CBT 治疗全流程操作（初访→评估→概念化→治疗→结束→督导）、治疗过程中的常见挑战与应对策略、新手治疗师实践指导。以珍妮案例贯穿全书，演示 CBT 从首次接触到结束的完整过程。

   b. **儿童青少年 CBT 适配**（若个案年龄未满 18 岁，须额外读取）：
      `./knowledge/干预/英文版_儿童与青少年认知行为疗法_CBT_for_children_and_adolescents/`
      先读 `*_index.md`（L1），按 Agent 使用建议读取发展适应性调整相关章节（L2）。该书籍用于补充：儿童青少年的认知发展水平对 CBT 技术的适配要求、家长参与策略、学校协作、发展性个案概念化调整、以及游戏化和具象化技术的应用。

   c. 其他维持机制 → 参考下方「干预技术识别与建议」模块匹配专项干预书籍。

生成时应优先体现：

- 主诉与问题清单；
- 触发情境、自动思维、情绪、生理反应、行为反应；
- 认知歪曲、补偿策略、回避和安全行为；
- 中间信念、核心信念、早期经验与当前问题的连接；
- 维持循环与保护因素；
- CBT 治疗目标、技术选择、家庭作业、进展评估和概念化修订。

## 精神分析或心理动力学取向

适用情况：

- 用户明确要求以精神分析、心理动力学、客体关系、自体心理学、依恋、冲突、防御、移情/反移情为主。
- 用户要求解释症状背后的无意识冲突、发展史、关系模式、防御方式和人格组织。

读取顺序：

1. **优先读取（L1 → L2）**：

   `./knowledge/个案概念化/英文版-心理动力学个案概念化Cabaniss_Psychodynamic_Formulation/`

   先读：
   `英文版-心理动力学个案概念化Cabaniss_Psychodynamic_Formulation_index.md`

   再根据任务需要读取对应章节文件：
   - 写心理动力学个案概念化 → `_01_03_总论与三步法.md`（了解 DESCRIBE/REVIEW/LINK 三步法）
   - 描述来访者功能 → `_04_08_描述功能.md`
   - 采集发展史 → `_09_12_回顾发展史.md`
   - 做动力学链接（创伤/冲突） → `_13_15_创伤困难与冲突防御.md`
   - 做动力学链接（关系/自体/依恋） → `_16_18_关系自体与依恋.md`
   - 特殊场景（急诊/药物/长程） → `_19_22_临床应用与实践.md`

2. **再读取理论补充（L1 → L2）**：

   `./knowledge/Counseling and Psychotherapy Theories in Context and Practice/`

   精神分析/心理动力学相关章节：
   `Counseling and Psychotherapy Theories in Context and Practice_02_03_精神分析与阿德勒治疗.md`

   该文件用于补充精神分析理论在咨询理论体系中的人性观、问题形成机制、治疗关系、技术选择，以及该章节内的"个案概念化模板"。

3. **干预技术补充**：如需动力学取向的干预指导，可读取：

   `./knowledge/干预/英文版-心理动力学疗法Psychodynamic Psychotherapy A clinical manual/`
   
   先读 `*_index.md`（如有），再读对应章节；该书为单文件提取，可直接按需检索。

生成时应优先体现：

- 当前症状与主诉；
- 重要发展史、依恋经验和关系创伤；
- 核心冲突、愿望、恐惧、羞耻、罪疚和自我体验；
- 防御方式及其适应性/代价；
- 重复性关系模式、移情和反移情线索；
- 人格功能、自体功能、现实检验、情绪调节和关系能力；
- 治疗焦点、治疗框架、解释深度、支持性与表达性技术的比例。

## 诊断与鉴别

适用情况：

- 材料中呈现明确症状群，需要做诊断假设或鉴别方向。
- 用户要求 DSM-5 或 ICD-11 诊断意见。
- 治疗记录中已有诊断，需要理解该诊断的临床表现和鉴别要点。

读取顺序：

1. **确定使用的诊断系统**：
   - 用户未指定 → 默认优先使用 ICD-11（`./knowledge/诊断/ICD-11.../`）
   - 用户指定 DSM-5 → 使用 `./knowledge/诊断/DSM5中文版/`

2. **L1 → L2 读取**：

   先读对应的 `*_index.md`：
   - DSM-5：`DSM5中文版_index.md`
   - ICD-11：`ICD-11精神、行为与神经发育障碍临床描述与诊断指南  B5_index.md`

   再按症状群匹配对应章节文件：

   | 症状群 | DSM-5 章节文件 | ICD-11 章节文件 |
   |--------|---------------|-----------------|
   | 神经发育/儿童期 | `DSM-5_01_神经发育障碍与儿童期障碍.md` | ICD-11 对应章节 |
   | 精神病性/双相/抑郁 | `DSM-5_02_04_精神分裂症谱系_双相_抑郁障碍.md` | ICD-11 对应章节 |
   | 焦虑/强迫/创伤/分离 | `DSM-5_05_08_焦虑_强迫_创伤_分离障碍.md` | ICD-11 对应章节 |
   | 躯体症状/进食/睡眠/性 | `DSM-5_09_14_躯体症状_进食_排泄_睡眠_性功能_性别障碍.md` | ICD-11 对应章节 |
   | 物质使用/成瘾 | DSM-5 对应章节 | ICD-11 对应章节 |
   | 人格障碍 | DSM-5 对应章节 | ICD-11 对应章节 |

3. **诊断深度控制**：
   - 常规症状群识别和鉴别方向 → 停在 L2 章节文件（已含诊断标准概要）
   - 边界案例、诊断争议、用户追问精确诊断标准 → 进入 L3，读取 `book/` 原文

4. **使用边界**：诊断表述、证据要求和材料不足时的处理服从 `AGENTS.md` 与 `guides/临床工作流程.md`；本节只决定诊断知识文件的读取路由。

- 识别治疗记录中治疗师已经使用的干预技术。
- 根据个案概念化结果，为治疗师推荐后续干预方向。
- 用户询问特定干预技术的操作细节。

### 干预读取流程

1. 从个案概念化中提取核心维持机制。
2. 对上表，确定匹配的干预书籍（可能 1–3 本）。
3. 读取对应书籍的 `*_index.md`（L1），按 Agent 使用建议读章节文件（L2）。
4. 以 L2 知识生成干预建议。仅在需要精确技术步骤/脚本时进入 L3。

### 既往干预技术识别

识别治疗记录中已使用技术时，只能依据记录中的治疗师话语、任务布置、互动方式或会谈结构。常见识别线索：

- CBT 相关：议程设置、情境-想法-情绪-行为分析、认知重构、行为实验、暴露、行为激活、作业复盘、进展监测
- 心理动力学相关：移情解释、防御分析、自由联想、发展史探索、关系模式澄清
- DBT 相关：链锁分析、技能教练、日记卡、痛苦忍受练习、正念引导
- SFBT 相关：奇迹问题、例外问题、量尺问题、赞许、应对问题
- 家庭治疗相关：联结（joining）、家谱图、环形提问、改释（reframing）、互动模式识别、界限工作、家庭生命周期讨论、治疗性三角形
- 一般支持性：共情、澄清、总结、情感反映

若材料不足以识别具体技术，写"记录中未能明确识别"，不要强行归类。具体输出方式仍服从 `guides/临床工作流程.md`。
