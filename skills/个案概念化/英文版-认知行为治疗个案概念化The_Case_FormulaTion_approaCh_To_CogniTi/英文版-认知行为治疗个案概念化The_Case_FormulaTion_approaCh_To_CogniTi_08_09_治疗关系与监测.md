## 第 8 章：治疗关系

### 章节主线

- 本书反对将 CBT 技术与治疗关系对立起来——关系问题常决定技术能否实施，而技术也可以改善关系。
- 治疗关系在个案概念化驱动的 CBT 中扮演**三种角色**：作为实施技术的协作条件、作为观察来访者人际模式的样本、作为改变机制的一部分。
- 当治疗关系出现困难时，这不是治疗路上的障碍，而可能是最直接的治疗材料——治疗师应将其纳入概念化并与来访者共同处理。

### CBT 中治疗关系的双重视角

**视角 1：必要但不充分（Necessary-but-Not-Sufficient, NBNS）**

- 信任和协作关系是 CBT 技术干预的基础——没有良好的协作关系，暴露、认知重构、行为实验等技术难以有效实施。
- 但仅有好的关系是不够的——还需要具体的技术干预来实现症状改变。

**视角 2：作为评估与干预工具**

- 治疗关系本身可以作为评估资料（来访者的人际模式样本）和干预场域（在此直接处理核心问题）。
- 来访者在治疗关系中的行为往往是其在外部世界人际模式的**现场样本**。

### 协作关系（Collaborative Relationship）

- CBT 依赖协作——治疗师和来访者共同制定议程、检验想法、设计实验、监测进展和调整计划。
- 协作不是治疗师放弃专业判断，也不是来访者被动接受专家指令，而是**双方共同检验假设**。
- 协作关系受阻时，要把它当成临床数据而非抱怨：
  - 来访者不愿设议程→可能反映回避、自主性困难、对控制的敏感或害怕失败。
  - 来访者不断寻求保证→可能反映不确定性耐受低。
  - 来访者讨好治疗师→可能反映"必须满足别人才能被接纳"的图式。
  - 来访者攻击治疗师→可能反映威胁敏感、人际防御或治疗师的实际失误。

### 治疗关系是外部关系的样本

- 会谈中的互动往往是来访者治疗室外生活模式的**样本**。
- 治疗师要观察：
  - 来访者如何提出需要。
  - 如何回应不同意见。
  - 是否回避冲突。
  - 是否让治疗师替其负责。
  - 是否理想化、贬低、讨好、撤退或测试关系。
  - 是否把治疗师放在熟悉的旧角色中。

- **例：Chie**的"关闭和放弃"（shut down and give up）模式在治疗关系中表现为取消咨询或退出治疗。概念化帮助治疗师将其识别为核心问题行为的体现，而非"阻抗"。

### 使用治疗师的情绪反应

- 治疗师的情绪反应可以作为数据，但必须谨慎使用。
- 治疗师感到困惑、无助、被拉去拯救、被攻击、被控制、想回避或想过度解释时，可能反映了来访者的人际影响模式。

**使用原则**

1. 先自我觉察，不急着行动。
2. 区分治疗师个人反应和来访者普遍人际影响。
3. 把反应转化为假设，而非指责。
4. 必要时以合作方式向来访者提出。

**示例对话**

> "我注意到我们每次谈到作业，你都很快转到别的话题。我也发现自己开始想替你决定下一步。我想知道，这是否也发生在你和别人一起做困难任务的时候？"

### 治疗关系中的问题处理

**处理步骤**

1. 注意到关系中的不顺畅（觉察）。
2. 具体描述发生了什么（行为描述，不贴标签）。
3. 询问来访者的体验（"你刚才是什么感觉？"）。
4. 提出与概念化相关的假设（"这可能跟你的……想法有关？"）。
5. 和来访者共同检验（"你觉得这个解释合理吗？"）。
6. 将关系中的新理解迁移到外部生活（"这在你生活中其他关系里出现过吗？"）。

**重要提醒**：治疗师不能把所有关系问题都解释为来访者的模式。治疗师可能确实做错了、误解了、忽视了文化背景、过度推进了或没有充分解释计划。**要首先承认现实部分**（validating the realistic elements）。

### 技术和关系不是二选一

- 关系问题常决定技术能否实施。
- 技术也可以改善关系：议程设置、作业回顾、行为实验、认知重评都需要协作——如果关系中存在顺从、回避或敌意，技术就会被扭曲。
- **例：Adrienne**因治疗师未能及时回电话而暴怒攻击——概念化帮助治疗师将其愤怒理解为源于"被抛弃和被欺骗"的感受（来自其受虐经历而非治疗师的实际行为），通过讨论解决了关系问题。

### Agent 快速调用：关系卡住时的分析框架

当治疗关系困难时，按以下问题分析：

1. 当前关系困难的具体行为是什么？
2. 这是否在来访者其他关系中重复出现？
3. 这是否与个案概念化中的核心信念或维持机制一致？
4. 治疗师是否也贡献了问题？
5. 如何把关系问题转化为共同观察和实验？

## 第 9 章：进展监测

### 章节主线

- 进展监测是本书的核心章节之一——因为概念化是假设，治疗计划也是基于假设的行动方案，所以治疗师必须**用数据检验它们**。
- 监测提示治疗师和来访者：治疗不是"相信治疗师的方法"，而是"收集数据看什么方法对这个人有效"——这种态度本身就能增强治疗联盟和来访者参与度。
- 监测允许**早期识别非依从和失败**，在问题恶化前进行调整。

### 为什么要监测

- 判断症状是否改善。
- 判断目标是否推进。
- 判断维持机制是否改变。
- 发现治疗无效或恶化。
- 帮助治疗师和来访者保持共同焦点。
- 为调整概念化提供数据。
- 增强来访者的自我观察能力。

### 监测什么：结果与过程

**结果监测（Outcome Monitoring）**

- 症状严重度：抑郁、焦虑、强迫、创伤症状等。
- 功能：工作、学习、睡眠、社交、家庭。
- 目标进展：外出次数、完成任务、减少冲突。
- 风险：自杀意念、自伤、物质使用。

**过程监测（Process Monitoring）**

- 自动思维强度、信念相信度。
- 回避和安全行为频率。
- 活动水平（activity level）。
- 情绪强度和调节策略。
- 暴露练习完成情况。
- 家庭作业完成情况。
- 治疗联盟和治疗会谈质量。
- 关键机制是否变化。

### 标准化监测与个体化监测

**标准化量表（Standardized Measures）**

- 适合追踪常见症状：如抑郁（PHQ-9、BDI）、焦虑（GAD-7、BAI）、创伤（PCL-5）、强迫（OCI-R）。
- 优点：有常模可供比较、信效度已知、易于与研究和临床文献沟通。

**个体化指标（Idiographic Indicators）**

- 适合追踪来访者自己的目标和机制：如每天离床时间、与伴侣冲突次数、回避电话次数、暴食冲动强度、担忧时间、安全行为使用频率、核心信念相信程度。

**最佳做法**：通常两者结合——标准化量表提供可比较的症状趋势，个体化指标提供对当前概念化最有用的数据。

### 何时监测

监测可以发生在以下时间点：

- 每次会谈前（pre-session）。
- 每次会谈开始的 check-in。
- 会谈中，针对具体情绪或信念 rating。
- 会谈后，作为家庭作业。
- 每日记录。
- 治疗阶段转换时。
- 治疗结束和随访时。

**频率原则**：高风险、高波动或需要行为实验数据的问题，可能需要每日监测；稳定症状可每周或每次会谈监测。

### 如何使用监测数据（临床决策规则）

| 数据模式 | 可能的含义和行动 |
|----------|------------------|
| 症状改善 + 机制改变 | 概念化可能正确，继续巩固有效策略 |
| 症状改善 + 机制未变 | 检查改善是否来自其他因素（安慰剂、生活变化等） |
| 症状不变 | 检查概念化是否正确、干预是否足量、作业是否执行、环境是否阻碍 |
| 症状恶化 | 评估风险、治疗副作用、生活事件、干预是否过强 |
| 量表改善但目标没改善 | 治疗可能过度聚焦症状而忽视功能 |
| 目标改善但量表没改善 | 可能需要调整对进展的解释和治疗焦点 |

### 监测机制：不止问症状

个案概念化驱动的 CBT 不只问"症状有没有下降"，还问"假设中的机制有没有改变"。

**机制监测示例**

- 概念化认为抑郁由活动减少维持 → 监测活动量和愉悦/掌控感。
- 概念化认为社交焦虑由安全行为维持 → 监测安全行为和回避。
- 概念化认为失眠由担忧和不规律作息维持 → 监测睡眠、担忧时间和作息。
- 概念化认为自伤由情绪调节困难维持 → 监测情绪强度、冲动、技能使用和后果。

### 监测治疗过程

- 来访者是否理解治疗计划。
- 是否认同目标。
- 是否完成家庭作业。
- 会谈是否聚焦。
- 治疗联盟是否稳定。
- 来访者是否觉得治疗有帮助。
- 是否出现**隐性不依从**（表面同意但不执行）。

这些过程指标常常比症状变化更早提示治疗是否偏离轨道。

### 监测的临床态度

- 监测不是审查来访者表现，也不是证明治疗师正确——它是**共同收集数据**。
- 治疗师要避免把未改善理解为"来访者失败"，而应将其视为需要调整假设和计划的信号。

**推荐表述**

> "这些数据说明我们原来的假设可能还不完整。我们一起看看，到底是哪一部分没有被治疗计划覆盖。"

### 制定监测计划的模板（供 agent 使用）

一个完整的监测计划至少包括：

1. **结果指标**：症状、功能、风险。
2. **机制指标**：与概念化中维持因素直接相关的变量。
3. **过程指标**：作业、联盟、参与度、会谈质量。
4. **频率**：每日、每周、每次会谈或阶段性。
5. **决策规则**：什么变化意味着继续、调整、加强、暂停或转介。
